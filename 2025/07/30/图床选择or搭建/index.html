Backblaze B2 + Cloudflare Workers 图床搭建教程

为什么选择这个方案？

因为白嫖快乐<(￣︶￣)> ，虽然我现在在用的是服务器ヽ( ^ω^ ゞ )

准备工作

  1. 注册账号

    • Backblaze（需验证邮箱）
    • Cloudflare（需绑定域名）

  2. 安装工具

    • PicGo（图片上传客户端）

详细搭建步骤

第一步：配置Backblaze B2

1.1 创建私密桶

登录Backblaze控制台，进入B2 Cloud Storage：


  • 桶名称：der-a-test-bucket （桶名称是随意的）
  • 桶类型：选择Private（私密桶）
  • 地区：建议选择US West (Oregon)

创建后记录Endpoint（如我的是https://s3.us-west-004.backblazeb2.com）

点击 Lifecycle Settings ，选择 Keep only the last version of the file 。点击 Update Bucket 按钮。

第二步：创建应用密钥

进入Application Keys页面：


  • 密钥名称：chenderupadtstest（钥匙名称也是随意的）
  • 权限设置：勾选readandwrite
  • 桶访问：选择刚才创建的私密桶

⚠️ 重要：创建后立即复制keyID和applicationKey，页面关闭后无法再次查看

第二步：部署Cloudflare Workers

2.1 创建Worker

登录Cloudflare控制台，找到左侧菜单的计算(Workers)点击进入Workers和Pages：


  • 点击「创建」→「创建Worker」→「选择从 Hello World！开始 模板」
  • Worker 名称：test（可自定义）
  • 点击「部署」后进入编辑界面

2.2 配置Worker代码

项目部署好后，我们点击 编辑代码 按钮，将下面的代码替换掉左侧编辑区 worker.js 中的全部代码，然后点击右上角的部署即可

第三步：配置 Cloudflare Worker

部署成功后直接返回我们创建的 Worker 页面，依次点击设置-变量和机密-添加 。

我们依次添加下面5个变量，变量 名称 和 值 分别为：

  • ALLOW_LIST_BUCKET = false （这里的意思时是否允许输出文件目录，有要求也可以改成ture）
  • B2_APPLICATION_KEY = 第二步保存的 applicationKey
  • B2_APPLICATION_KEY_ID = 第二步保存的 keyID
  • B2_ENDPOINT = 创建桶时你的桶的 Endpoint 值（如我的是 s3.us-east-005.backblazeb2.com）
  • BUCKET_NAME = 私密桶名（如我的是 der-a-test-bucket）

第四步：配置自定义域名访问

同样在设置，点击 域和路由-添加 ，选择自定义域

提示：自定义域名生效需要时间，大概需要5分钟到1个小时不等。

第五步：配置 Backblaze B2 私密桶

我们回到 Backblaze ，点击左侧一栏Bucket，定位到我们刚刚新建的私密桶，点击 Bucket Settings ：

在 Bucket Info 项填入：{“Cache-Control”: “public, max-age=86400”}
最后点击 Update Bucket 按钮即可。

第六步：桶访问测试

6.1 上传图片到私密桶

我们再回到新建的私密桶，点击 Upload/Download 按钮进入桶，然后点击 Upload 按钮上传一个图片（随意什么图片，用来测试而已）。

6.2 查看图片的 URL

我们直接点击我们上传到桶里面图片，查看图片

6.3 访问图片

然后我们直接访问图片的 S3 URL （https://http://der-a-test-bucket.s3.us-east-005.backblazeb2.com/zhuzhuxiamenu.png） ，可以发现我们是看不到图片的。

而当我们访问我们的worker（https://test.2876563462.workers.dev/zhuzhuxiamenu.png）或者我们配置的自定义域名时（https://你的自定义域名/zhuzhuxiamenu.png）则可以看到图片。（⚠️注意：在你的自定义域名设置好前，你是可以用worker自带的域名访问的，但在你的自定义域名设置好后，你就无法用worker的自带域名访问图片了）

提示：若你的结果符合上述则说明 Cloudflare Worker 已经生效了。

第七步：配置 PicGo

这里PicGo可以搭配Typora一起使用，这样就可以通过复制图片的方式直接上传到图床。方法很简单，有兴趣可以看看下面的文章。
PicGo搭配Typora

7.1 安装 S3 插件

点击左侧 插件设置 ，搜索 S3 ，这里 s3 和 s3-lls 都可以，我选择的是 s3-lls ，点击安装。

7.2 配置 Amazon S3 图床

点击 图床设置 - Amazon S3 - 编辑 ，配置详情如下：

配置完后直接点击 确定 按钮即可。
注：上传路径可参考文件 PicGo S3 插件 内规则，我在上传前都会把图片按日期和文章命名好，你也可以选择默认的 {year}/{month}/{fullName}

7.3 上传测试

点击左侧 上传区 ，尝试上传图片。

上传后可点击左侧 相册 和回到我们的桶查看，均显示图片已上传。

到这里我们的 cf＋b2 图床就搭建好了。

需要注意的问题

cloudflare的免费计划是有上限的
A：cloudflare每天的免费请求只有100,000次，用超了如果你没有绑卡就会停，绑了卡就会自动扣费，即便超了价格还是很亲民的。

PicGo上传失败
A：注意A3的配置里面的权限一定要填 private ，默认的 public-read Backblaze桶是识别不出来的
